- **Pre-Step Checklist** in `state.md`: reset all boxes `[ ]`, then check each `[x]` as completed before starting the step. These boxes are an advisory re-read aid, not a mechanically enforced gate ‚Äî no agent ticks them automatically; the executor's own Pre-Step Checklist (ip-executor.md) and the orchestrator's Post-Step Gate perform and confirm the actual reads.
- Iteration 1, first EXECUTE ‚Üí create `checkpoints/cp-000-iter1.md` (nuclear fallback). "Git State" = commit hash BEFORE changes (the restore point).
- One step at a time. Post-Step Gate after each (see below).
- Checkpoint before risky changes (3+ files, shared modules, destructive ops). Name: `cp-NNN-iterN.md` (e.g. `cp-001-iter2.md`). Increment NNN globally across iterations.
- Commit after each successful step: `[plan-YYYY-MM-DD-HASH/iter-N/step-M] description`.
  - **Deriving the tag id**: take the plan-dir name and **drop the `THHMMSS` segment**. `plan-2026-07-14T051317-317362c4` ‚Üí `[plan-2026-07-14-317362c4/iter-3/step-2] description`. A **legacy** plan dir (`plan_YYYY-MM-DD_XXXXXXXX` ‚Äî plans created before v2.36.0 are still being executed) derives identically, normalizing the `_` separators to `-`: `plan_2026-07-14_79ee0f59` ‚Üí `[plan-2026-07-14-79ee0f59/iter-3/step-2] description`.
  - **The changelog `step` field stays bare `iter-N/step-M`** ‚Äî do not "fix" this apparent inconsistency. That field is sourced from `state.md`, never parsed from the commit subject, and nothing in the codebase reads a commit message. Prefixing it would drag in `schema.mjs` / `STEP_RE` and the compression `from`/`to` range bounds for zero benefit.
- If something breaks ‚Üí STOP. 2 fix attempts max (Autonomy Leash). Each must follow Revert-First.
- **Irreversible operations** (DB migrations, external API calls, service config, non-tracked file deletion): mark step `[IRREVERSIBLE]` in `plan.md` during PLAN. Full procedure: `references/code-hygiene.md`.
- **Surprise discovery** (behavior contradicts findings, unknown dependency, wrong assumption, or an existing `# DECISION` anchor whose rationale contradicts this step's approach) ‚Üí check plan.md Assumptions to identify which steps are invalidated. Note in `state.md`, finish or revert current step, transition to REFLECT. Do NOT silently update findings during EXECUTE.
- **Falsification signal fires** (from Pre-Mortem & Falsification Signals in plan.md) ‚Üí same as surprise discovery. Log which signal fired in `decisions.md`.
- Add `# DECISION <plan-id>/D-NNN` comments (e.g. `# DECISION plan-2026-05-07T091743-7556fb98/D-003`; a legacy `plan_2026-05-07_7556fb98/D-003` prefix stays valid on plans created before v2.36.0) where any of the 5 trigger conditions in `references/decision-anchoring.md` apply. Plan-id prefix matches the active plan directory name.
- **Python/software tasks**: for Python or software-engineering work, consult `references/python-software.md` ¬ß B.16 When NOT to apply these patterns before adding any structure (most code should NOT reach for a pattern), and its Python style + anti-patterns section while writing ‚Äî the 20-item ¬ß C.12 Anti-pattern checklist is the REFLECT review gate this code will be graded against. Skip for non-software plans.
- **Per-Edit Changelog (v2.15.0+)**: after each `Edit` or `Write`, append one pipe-delimited line per file to `{plan-dir}/changelog.md` recording timestamp, iter/step, commit (or `uncommitted`), path, op + LOC, blast-radius (`node <skill-path>/scripts/blast-radius.mjs <file>` ‚Äî first stdout line), decision-ref (`D-NNN` or `-`), and a one-clause reason. Append-only. Decision-ref is optional ‚Äî `-` is fine for most edits. The 5 `# DECISION` trigger conditions are unchanged. Radius script always exits 0 ‚Äî never blocks the step. See `references/file-formats.md` for full format (or run `node <skill-path>/scripts/emit-template.mjs --name changelog` to get just this template ‚Äî file-formats.md is the canonical fallback) and `references/blast-radius.md` for radius scoring.

#### Post-Step Gate (successful steps only ‚Äî all 4 before moving on)
1. `plan.md` ‚Äî mark step `[x]`, advance marker, update complexity budget
2. `progress.md` ‚Äî move item Remaining ‚Üí Completed, set next In Progress
3. `state.md` ‚Äî update step number, append to change manifest
4. `changelog.md` ‚Äî confirm one line per file edited in this step (validator WARNs on drift)

**After the gate, before the next step**: run `node <skill-path>/scripts/bootstrap.mjs reset-attempts` to clear the Fix Attempts section (resets the whole section to placeholder). The pre-step gate counts attempt lines section-wide (not per-step), so a stale counter from a step that used ‚â•1 attempt then succeeded would spuriously HARD-trip `leash-cap` on the next step (SKILL.md Autonomy Leash ‚Äî "Resets on: user direction | new step | PIVOT"). This is the operative wiring of that spec'd reset; the same command also runs on PIVOT (state-pivot.md).

On **failed step**: skip gate. Follow Autonomy Leash (revert-first, 2 attempts max).
